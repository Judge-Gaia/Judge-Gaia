# Judge Gaia Development Docs

이 디렉터리는 개발 중 방향을 잃지 않기 위한 기준 문서 모음이다.

## 문서 구성

- `01-project-brief.md`: 프로젝트 정체성, 목표, 타깃, 표현 원칙.
- `02-product-requirements.md`: 전체 게임 플로우와 주요 시스템 요구사항.
- `03-architecture-stack.md`: 현재 선택한 기술 스택과 패키지 책임.
- `04-week-01-mvp-plan.md`: 이번 세션/1주차 구현 목표와 완료 조건.
- `05-acceptance-criteria.md`: MVP 수용 기준과 검증 관점.
- `reports/interim-report-01-guide.md`: 중간 보고서 1 작성 가이드.

## 현재 개발 우선순위

1. 홈 화면에서 닉네임 입력 후 즉시 개인 세션 입장.
2. 화면 중앙 3D 지구 렌더링, 드래그 회전, 확대/축소.
3. Socket.IO 연결 시 서버가 고유 세션 ID를 발급.
4. PostgreSQL 유저/랭킹 스키마와 서버 연결 유지.
5. 중간 보고서용 시각 자료와 기능 요약을 남길 수 있는 상태 확보.
