# Acceptance Criteria

## MVP 전체 수용 기준

1. 유저가 30초 내 플레이에 진입할 수 있다.
2. 매칭 대기 없이 즉시 개인 세션이 시작된다.
3. 이벤트 클릭 후 2초 내 현장 전환이 시작된다.
4. 이벤트 1개 완료 시 업적 카드가 정상 노출되고 저장된다.
5. 플레이 종료 시 점수와 랭킹은 서버 기준으로 일관된다.
6. 중저사양 환경에서도 FPS 30 이상으로 플레이가 지속된다.

## 1주차 수용 기준

1. `pnpm dev`로 프론트와 서버가 함께 실행된다.
2. 프론트 홈 화면에서 3D 지구가 중앙에 렌더링된다.
3. 사용자가 지구를 드래그해 회전할 수 있다.
4. 사용자가 지구를 확대/축소할 수 있다.
5. 닉네임 입력 후 즉시 입장하면 서버에서 세션 ID를 받는다.
6. 세션 ID가 클라이언트 상태에 저장되고 화면에 반영된다.
7. 서버는 PostgreSQL 연결 설정과 유저/랭킹 스키마를 갖춘다.
8. 프론트엔드/백엔드 핵심 코드에 보고서용 주석이 있다.

## 검증 명령

```bash
pnpm typecheck
pnpm lint
pnpm build
```

## 수동 검증

- `http://localhost:5173` 접속.
- 닉네임 입력.
- 즉시 입장 클릭.
- 화면에 개인 세션 ID 표시 확인.
- 3D 지구 드래그 회전 확인.
- 확대/축소 확인.
- 서버 로그에서 Socket.IO 입장 이벤트 확인.
