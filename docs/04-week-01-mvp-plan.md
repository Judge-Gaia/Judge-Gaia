# Week 01 MVP Plan

## 목표

3일 안에 프론트엔드와 백엔드의 뼈대를 연결하고, 사용자가 닉네임 입력 후 3D 지구 홈 화면에서 개인 세션으로 즉시 입장할 수 있는 상태를 만든다.

진행률 기준: 1주차 완료 시 전체 MVP의 20%.

## 클라이언트 & 3D 개발 목표

- React + Vite 환경 세팅.
- Three.js 초기화.
- 화면 중앙에 3D 지구 객체 렌더링.
- 마우스 드래그로 지구 회전.
- 휠 또는 트랙패드로 확대/축소.
- 닉네임 입력 UI.
- 즉시 입장 버튼.
- 개인 세션 진입 상태 표시.
- 홈 화면에 SDGs 13/14/15 메시지 포함.

## 백엔드 & DB 개발 목표

- Node.js + Fastify 서버 세팅.
- PostgreSQL 연결.
- 유저 스키마 생성.
- 랭킹 스키마 생성.
- Socket.IO 연결.
- 클라이언트 입장 시 고유 세션 ID 발급.
- 닉네임 기반 게스트 유저 생성 또는 세션 메타데이터 저장.

- [x] React + Vite 프로젝트 구성.
- [x] Three.js 초기 렌더링 구성.
- [x] Fastify 서버 구성.
- [x] PostgreSQL 초기 스키마 추가.
- [x] Socket.IO 기본 연결 구성.
- [x] 3D 지구 드래그 회전/확대 인터랙션.
- [x] 닉네임 입력 홈 화면 UI.
- [x] 입장 요청 Socket 이벤트.
- [x] 서버 세션 ID 발급 이벤트.
- [x] 세션 시작 후 클라이언트 상태 반영.
- [x] SDGs 메시지 UI 반영.
- [x] 프론트엔드/백엔드 핵심 코드 주석 보강.
- [ ] 3D 홈 화면 캡처.

## 완료 조건

- 사용자가 30초 내 닉네임을 입력하고 개인 세션에 들어갈 수 있다.
- 매칭 대기 없이 서버가 세션 ID를 발급한다.
- 홈 화면 중앙에서 3D 지구가 보인다.
- 지구는 드래그로 회전하고 확대/축소할 수 있다.
- 서버 `/health`와 Socket.IO 입장 이벤트가 모두 동작한다.
- 중간 보고서 1에 넣을 홈 화면 캡처를 만들 수 있다.

## 코드 주석 기준

보고서 요구사항 때문에 다음 위치에는 짧은 설명 주석을 유지한다.

- Three.js 장면/카메라/렌더러 초기화.
- 마우스 드래그 회전과 확대/축소 입력 처리.
- Socket.IO 입장 이벤트 송신/수신.
- 서버의 세션 ID 발급 로직.
- DB 유저/랭킹 스키마 또는 저장 함수.
